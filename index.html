Подробная справка по языку UPC

Что такое UPC?
UPC (Universal Patch Container) — это упрощённый язык для написания патчей к программам.
С его помощью можно создавать скрипты, которые встраиваются в программы и изменяют их поведение, файлы или содержимое.
Язык разработан для удобства и универсальности.

Основные команды UPC

patch — объявляет начало патча и его версию.
пример:
patch 1.0
Описывает патч версии 1.0.
description — описание патча (текст).
пример:
description "исправление ошибки загрузки"
replace "файл.file", "старый_текст" to "новый_текст"
Заменяет в указанном файле все вхождения старого_текста на новый_текст.
Если файл не указан, замена происходит в основном файле патча (если такой есть).
пример:
replace "config.ini", "Hello" to "Привет"
delete "файл.file" "текст"
Удаляет указанную строку из указанного файла.
Если файл не указан, удаление происходит в основном файле патча.
пример:
delete "config.ini" "debug_mode=1"
Или
delete "debug_mode=1" — удаление из основного файла.
copy источник to назначение
Копирует файл.
пример:
copy "backup.ini" to "config.ini"
create путь_к_файлу
Создаёт новый файл с содержимым, записанным в блоке.
пример:
create "new_script.py"
(
    print("привет мир")
    print("это новый скрипт")
)
ce
Примечание: ce — команда окончания блока создания.
embed [(create)] "имя_файла" type base64
Встраивает бинарный файл в патч в формате base64.
Если указан флаг create, то создаёт этот файл при применении патча.
пример:
embed create "image.png" type base64
(
iVBORw0KGgoAAAANSUhEUgAA...
)
ce
exec_if условие ( блок_команд )
Выполняет блок команд, если условие истинно.
пример:
exec_if file_exists "plugin.dll" (
  message "плагин уже установлен"
  stop
)

Пример полного патча

patch 1.0
description "пример расширенного патча с заменой, созданием и embed"

replace "config.ini", "Hello" to "Привет"

create "hello.py"
(
print("привет мир!")
)
ce

embed create "image.png" type base64
(
iVBORw0KGgoAAAANSUhEUgAA...
)
ce

message "патч успешно применён!"

exec_if file_exists "plugin.dll" (
  message "плагин уже установлен"
  stop
)

Советы и рекомендации

  • Пишите команды с маленьких букв — это облегчает чтение и парсинг.
  • Текст внутри кавычек "" — чувствителен к регистру и должен быть точным.
  • Для больших файлов используйте команду embed с base64, чтобы встроить бинарные данные.
  • Всегда проверяйте синтаксис перед применением патча.
  • Используйте backup, чтобы сохранить оригиналы файлов перед изменениями.
  • Проверяйте условия в if и exec_if, чтобы избежать ошибок при применении патчей.
